A lot of companies still rely on cron, the Unix utility for time-based job scheduling. It's simple to set up but it doesn't scale; it's error-prone; and it's hard to debug.

Authentication

Requests to all endpoints have to be authenticated with a Authorization: Bearer {token} header. See the example below for more details. Email {{ crono_support_email }} to request a token.

Example 

POST {{ reverse_url('post_job') }} HTTP/1.1
Host: {{ crono_base_url }}
Content-Type: application/json; charset=utf-8
Authorization: Bearer {{ crono_api_token }}
